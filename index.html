          Firebase Connection

          Checking connection...

          Edit the firebaseConfig object near the top of the script section of this file to connect your Firebase project, then reload. Until then, data is saved locally in this browser.

          If nothing seems to respond / no data loads: your Firestore Database may not be created yet, or its security rules are blocking access. In the Firebase Console → Firestore Database → Rules, use:

          rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

          This matches the same open, no-auth, no-expiry rule pattern used on your other JO/EMR systems — access is controlled client-side by the Edit PIN, not by Firebase Auth.
